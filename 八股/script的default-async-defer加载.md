# script 标签的默认/async/defer 三种加载

无论哪种情况，浏览器都是碰到了就立即开始请求，但是执行的时候会不一样

并且一般都是按顺序执行，一般都是脚本执行时均为阻塞

现代推荐 `defer`

## 默认情况

浏览器会暂停 HTML 的解析，阻塞式请求这个资源，请求完后立即执行

### 关于为什么阻塞

为了防止脚本修改尚未构建的 DOM，确保脚本 JS 执行时面对的 DOM 环境是固定的、可预测的

### 关于 CSS 加载阻塞 script

原因类似，防止脚本修改未完成的 CSS 样式

反正脚本最底层，因为脚本会动态修改很多东西

## async 情况

浏览器不会暂停 HTML 的解析，而是并行进行请求和解析 HTML 的工作，请求完后立即阻塞式执行

## defer 情况

defer 意为推迟，推迟执行

浏览器不会暂停 HTML 的解析，而是并行进行请求和解析 HTML 的工作，请求完成后，HTML 解析完成后才执行
